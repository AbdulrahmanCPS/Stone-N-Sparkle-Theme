# Security policy

## Supported versions

We provide security-related updates for the current theme version in use on our production site. Older versions are not supported.

## Reporting a vulnerability

If you believe you have found a security vulnerability in this theme:

1. **Do not** open a public GitHub issue.
2. Email the maintainers (or the contact you have for the Stone & Sparkle project) with:
   - A clear description of the issue
   - Steps to reproduce
   - Impact and suggested fix if you have one
3. We will acknowledge receipt and aim to respond within a reasonable time. We may ask for more detail.
4. We will work on a fix and coordinate with you on disclosure (e.g. release note, CVE) if applicable.

Please do not disclose the vulnerability publicly until we have had a chance to address it.
